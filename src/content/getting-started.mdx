# Getting Started

Creddy issues short-lived credentials to AI agents. Your agents never see your master secrets — they authenticate to Creddy and get temporary, scoped tokens.

**Architecture:**
```
┌──────────────┐         ┌──────────────┐         ┌──────────────┐
│    Agent     │ ──────▶ │    Creddy    │ ──────▶ │   GitHub     │
│  (your bot)  │  auth   │   Server     │  master │   AWS, etc   │
│              │ ◀────── │              │  creds  │              │
│              │  temp   │              │         │              │
│              │  token  │              │         │              │
└──────────────┘         └──────────────┘         └──────────────┘
```

Get it running in 5 minutes.

---

## 1. Install Creddy

On your server (where master credentials will live):

```bash
curl -fsSL https://get.creddy.dev/install.sh | sudo sh
```

## 2. Start the Server

```bash
# Basic (local access only)
creddy server

# With OIDC enabled (recommended for production)
creddy server --oidc-issuer https://creddy.example.com
```

Output:
```
Database: /root/.creddy/creddy.db
OIDC Issuer: https://creddy.example.com
Listening on 127.0.0.1:8400
```

> **Production:** See [Server Setup](/docs/server) for TLS, systemd, and Tailscale configuration.

## 3. Add a Backend

Install the GitHub plugin:

```bash
creddy plugin install github

creddy backend add github \
  --app-id YOUR_APP_ID \
  --private-key /path/to/private-key.pem \
  --installation-id YOUR_INSTALLATION_ID
```

> **Need a GitHub App?** See [GitHub Integration](/docs/integrations/github).

## 4. Create an Agent

```bash
creddy agent create my-agent --scopes "github:owner/repo"
```

Output:
```json
{
  "id": "a1b2c3d4",
  "name": "my-agent",
  "token": "ckr_abc123...",
  "oidc": {
    "client_id": "agent_f8e7d6",
    "client_secret": "cks_xyz789..."
  },
  "scopes": ["github:owner/repo"]
}
```

**Save these credentials!** They're only shown once.

## 5. Use Credentials (Agent Side)

### Option A: OIDC Flow (Recommended)

```bash
# Set these in your agent's environment
export CREDDY_URL="https://creddy.example.com"
export CREDDY_CLIENT_ID="agent_f8e7d6"
export CREDDY_CLIENT_SECRET="cks_xyz789..."

# 1. Get access token
ACCESS_TOKEN=$(curl -s -X POST $CREDDY_URL/oauth/token \
  -d "grant_type=client_credentials" \
  -d "client_id=$CREDDY_CLIENT_ID" \
  -d "client_secret=$CREDDY_CLIENT_SECRET" \
  | jq -r .access_token)

# 2. Get GitHub credential
GITHUB_TOKEN=$(curl -s "$CREDDY_URL/v1/credentials/github?ttl=10m" \
  -H "Authorization: Bearer $ACCESS_TOKEN" \
  | jq -r .token)

# 3. Use it
gh api repos/owner/repo --header "Authorization: token $GITHUB_TOKEN"
```

### Option B: Legacy Flow (Simpler)

```bash
# Set this in your agent's environment
export CREDDY_TOKEN="ckr_abc123..."

# Get GitHub credential directly
GITHUB_TOKEN=$(curl -s "$CREDDY_URL/v1/credentials/github?ttl=10m" \
  -H "Authorization: Bearer $CREDDY_TOKEN" \
  | jq -r .token)
```

---

## What Just Happened?

1. **Server** holds your GitHub App credentials (the master secret)
2. **Agent** authenticated to Creddy (via OIDC or legacy token)
3. **Creddy** issued a 10-minute GitHub token scoped to `owner/repo`
4. **Agent** used the temporary token — it auto-expires, no cleanup needed

The agent never saw your GitHub App private key.
