# Creddy

**Ephemeral credentials for AI agents.**

Your AI agents need access to GitHub, AWS, and other services. But sharing your API keys is risky. Creddy solves this: agents authenticate to Creddy and get short-lived, scoped tokens. They never see your master credentials.

## The Problem

When you give an agent access to your systems:

1. **Share your credentials** — Agent has full access, forever ❌
2. **Create long-lived API keys** — Still risky if compromised ❌
3. **Use Creddy** — Agent gets 10-minute scoped tokens ✅

## How It Works

```
┌──────────────┐         ┌──────────────┐         ┌──────────────┐
│    Agent     │ ──────▶ │    Creddy    │         │   GitHub     │
│  (your bot)  │  auth   │   Server     │         │   AWS, etc   │
│              │ ◀────── │              │ ◀────── │              │
│              │  temp   │  (holds      │  master │              │
│              │  token  │   secrets)   │  creds  │              │
└──────────────┘         └──────────────┘         └──────────────┘
```

1. **Agent authenticates** to Creddy (via OIDC or token)
2. **Creddy issues** a short-lived credential for the requested backend
3. **Agent uses** the credential (expires automatically)

The agent never sees your GitHub App private key or AWS root credentials.

## Key Features

- 🔐 **Ephemeral** — Credentials expire in minutes, not months
- 🎯 **Scoped** — Grant only what's needed (`github:owner/repo:read`)
- 📊 **Auditable** — Track every credential issued
- 🌐 **OIDC Provider** — Standard OAuth 2.0, works with AWS/GCP federation
- 🔌 **Plugin System** — GitHub, AWS, Doppler, Anthropic, and more

## Quick Example

**Server (holds your secrets):**
```bash
creddy server --oidc-issuer https://creddy.example.com
creddy backend add github --app-id 123 --private-key ./key.pem
creddy agent create my-bot --can github:owner/repo
```

**Agent (gets temporary credentials):**
```bash
# Authenticate
TOKEN=$(curl -s -X POST https://creddy.example.com/oauth/token \
  -d "grant_type=client_credentials" \
  -d "client_id=$CLIENT_ID" \
  -d "client_secret=$CLIENT_SECRET" \
  | jq -r .access_token)

# Get 10-minute GitHub token
GITHUB=$(curl -s "https://creddy.example.com/v1/credentials/github?ttl=10m" \
  -H "Authorization: Bearer $TOKEN" \
  | jq -r .token)

# Use it
gh api repos/owner/repo --header "Authorization: token $GITHUB"
```
