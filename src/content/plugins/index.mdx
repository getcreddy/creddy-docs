# Plugins

Creddy uses a plugin architecture for credential backends. Each integration (GitHub, Anthropic, Doppler, etc.) is a separate binary that communicates with Creddy core via gRPC.

## Why Plugins?

- **Independent releases** — Plugins can be updated without updating Creddy core
- **Smaller core** — Only install the integrations you need
- **Community extensibility** — Anyone can build and share plugins
- **Private integrations** — Build internal plugins for proprietary systems

## Discovering Plugins

List available plugins from the official registry:

```bash
creddy plugin list
```

Output:
```
NAME        VERSION   DESCRIPTION                                      INSTALLED
github      0.2.1     GitHub App installation tokens                   ✓ 0.2.1
anthropic   0.1.3     Anthropic API keys (Admin API)                   ✓ 0.1.3
doppler     0.1.2     Doppler service tokens                           -
aws         0.1.0     AWS IAM temporary credentials                    -
```

Search for plugins:

```bash
creddy plugin search docker
```

## Installing Plugins

Install from the official registry:

```bash
# Install latest version
creddy plugin install github

# Install specific version
creddy plugin install github@0.2.0

# Install multiple plugins
creddy plugin install github anthropic doppler
```

Install from a custom URL (for private/third-party plugins):

```bash
creddy plugin install https://internal.corp/plugins/salesforce
creddy plugin install https://github.com/someone/creddy-custom/releases/download/v0.1.0/creddy-custom-linux-amd64.tar.gz
```

## Using Plugins

Once installed, plugins are automatically available. Add a backend:

```bash
# GitHub plugin
creddy backend add github \
  --app-id 123456 \
  --installation-id 78901234 \
  --private-key /path/to/app.pem

# Doppler plugin  
creddy backend add doppler \
  --token "dp.st.xxx"
```

Request credentials:

```bash
# Uses the github plugin
creddy get github --scope "github:myorg/myrepo"

# Uses the doppler plugin
creddy get doppler --scope "doppler:myproject/production"
```

## Upgrading Plugins

Check for updates:

```bash
creddy plugin outdated
```

Output:
```
NAME        INSTALLED   LATEST    
github      0.2.0       0.2.1     
doppler     0.1.1       0.1.2     
```

Upgrade plugins:

```bash
# Upgrade specific plugin
creddy plugin upgrade github

# Upgrade all plugins
creddy plugin upgrade --all
```

## Managing Plugins

View installed plugins:

```bash
creddy plugin list --installed
```

Get plugin details:

```bash
creddy plugin info github
```

Output:
```
Name:              github
Version:           0.2.1
Description:       GitHub App installation tokens with repository scoping
Min Creddy Version: 0.4.0
Installed:         ~/.creddy/plugins/creddy-github
Source:            https://plugins.creddy.dev/github/0.2.1

Scopes:
  github:*              All repositories the App is installed on
  github:<owner>/*      All repositories under an owner/org
  github:<owner>/<repo> Specific repository (add :read or :write)
```

Remove a plugin:

```bash
creddy plugin remove github
```

## Plugin Locations

Plugins are stored in `~/.creddy/plugins/` by default:

```
~/.creddy/
  plugins/
    creddy-github
    creddy-anthropic
    creddy-doppler
```

Override with environment variable:

```bash
export CREDDY_PLUGIN_DIR=/opt/creddy/plugins
```

## Troubleshooting

### Plugin not found

```bash
# Check if plugin is installed
creddy plugin list --installed

# Reinstall
creddy plugin remove github
creddy plugin install github
```

### Plugin version mismatch

```
Error: plugin github requires Creddy >= 0.5.0 (you have 0.4.2)
```

Upgrade Creddy or install an older plugin version:

```bash
creddy plugin install github@0.1.5
```

### Debug plugin communication

```bash
CREDDY_PLUGIN_DEBUG=1 creddy get github --scope "github:myorg/repo"
```
