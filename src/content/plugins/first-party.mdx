## github

GitHub App installation tokens with repository scoping.

**Repository:** [getcreddy/creddy-github](https://github.com/getcreddy/creddy-github)

### Installation

```bash
creddy plugin install github
```

### Configuration

```bash
creddy backend add github \
  --app-id 123456 \
  --installation-id 78901234 \
  --private-key /path/to/app.pem
```

### Scopes

| Scope | Description |
|-------|-------------|
| `github:*` | All repos the App is installed on |
| `github:owner/*` | All repos under an owner/org |
| `github:owner/repo` | Specific repository |
| `github:owner/repo:read` | Read-only access |
| `github:owner/repo:write` | Write access (default) |

### Releases

#### v0.2.1 (Latest)
- Fixed: Token scoping for repos with special characters
- Fixed: Installation discovery when multiple orgs installed

#### v0.2.0
- Added: Repository-level scoping (`github:owner/repo`)
- Added: Read/write permission modifiers (`:read`, `:write`)
- Added: Owner wildcard support (`github:owner/*`)

#### v0.1.0
- Initial release
- Basic installation token generation

## anthropic

Ephemeral Anthropic API keys via the Admin API.

**Repository:** [getcreddy/creddy-anthropic](https://github.com/getcreddy/creddy-anthropic)

### Installation

```bash
creddy plugin install anthropic
```

### Configuration

```bash
creddy backend add anthropic \
  --admin-key "sk-admin-xxx"
```

### Scopes

| Scope | Description |
|-------|-------------|
| `anthropic:*` | Full API access |
| `anthropic:messages` | Messages API only |
| `anthropic:messages:read` | Read-only (completions) |

### Requirements

- Anthropic Scale or Enterprise plan (Admin API access required)

### Releases

#### v0.1.3 (Latest)
- Added: External ID tracking for revocation
- Fixed: TTL handling for keys > 24h

#### v0.1.2
- Added: Scope-based permission levels
- Improved: Error messages for API failures

#### v0.1.1
- Fixed: Key cleanup on agent unenrollment

#### v0.1.0
- Initial release
- Basic API key generation and revocation

## doppler

Scoped Doppler service tokens.

**Repository:** [getcreddy/creddy-doppler](https://github.com/getcreddy/creddy-doppler)

### Installation

```bash
creddy plugin install doppler
```

### Configuration

```bash
creddy backend add doppler \
  --token "dp.st.xxx"
```

### Scopes

| Scope | Description |
|-------|-------------|
| `doppler:project/config` | Access to specific project/config |
| `doppler:project/config:read` | Read-only access |
| `doppler:project/*` | All configs in a project |

### Releases

#### v0.1.2 (Latest)
- Added: Config wildcard support (`doppler:project/*`)
- Fixed: Token revocation on TTL expiry

#### v0.1.1
- Added: Read-only scope modifier
- Improved: Validation error messages

#### v0.1.0
- Initial release
- Basic service token generation
